Wireless and Mobile Networks

A comprehensive guide to understanding the principles, technologies, and challenges of wireless communication and mobility.

Summary and Basic Definitions

This study guide offers a comprehensive overview of Wireless and Mobile Networks, covering principles, characteristics, architecture, and management. It distinguishes between wireless communication (radio links) and mobility (changing network attachment points). Technologies discussed include CDMA, IEEE 802.11 (WiFi), Bluetooth, and Cellular Networks (2G, 3G, 4G LTE, 5G), alongside Mobility Management and Mobile IP.

Basic Definitions:

Wireless Host
Device communicating over a wireless link (e.g., laptop, smartphone), can be stationary or mobile.
Base Station (BS) / Access Point (AP) / eNodeB
Device connected to a wired network, relays packets between wired and wireless hosts in its coverage area.
Wireless Link
Communication medium using radio signals between mobile devices and base stations or other wireless nodes.
Mobility
Ability of a user/device to change network attachment points while maintaining ongoing connections.
Handoff
Process where a mobile device changes its base station connection, ideally without interrupting communication.
Infrastructure Mode
Wireless network where hosts connect to a wired network via a base station (AP).
Ad-Hoc Mode
Wireless network without base stations; nodes self-organize and route traffic among themselves.
Coverage Area
Geographical region with reliable wireless signal reception.
Signal-to-Noise Ratio (SNR)
Measure of signal strength relative to background noise; higher SNR aids signal extraction.
Bit Error Rate (BER)
Ratio of bit errors to total bits transferred; inversely related to SNR.
Chipping Sequence (Code)
Unique binary sequence in CDMA for encoding data, enabling simultaneous transmission with minimal interference.
Home Location Register (HLR)
Cellular home network database storing permanent user info and current location.
Visitor Location Register (VLR)
Cellular visited network database storing temporary info about users in that network.
Permanent Address
IP address assigned to a mobile device in its home network.
Care-of-Address (COA)
Temporary IP address assigned to a mobile device in a visited network.
Home Agent (HA)
Entity in the mobile's home network that intercepts packets for the mobile and tunnels them to its COA.
Foreign Agent (FA)
Entity in a visited network performing mobility functions for a mobile, receiving tunneled packets from the HA.
Correspondent
Node wishing to communicate with a mobile node.
Tunneling
Encapsulating one network protocol within another, used in Mobile IP to forward packets to a mobile's COA.

Key Concepts

Introduction to Wireless and Mobile Networks

  • Wireless (mobile) phone subscribers exceed wired phone subscribers (5-to-1 ratio).
  • Wireless Internet-connected devices equal wired Internet-connected devices.
  • Two main challenges: Wireless (communication over wireless link) and Mobility (handling users changing network attachment).
  • Wireless hosts (laptops, smartphones) run applications and can be stationary or mobile.
  • A base station (cell tower, 802.11 AP) connects wireless hosts to the wired network.
  • A wireless link connects mobiles to base stations, with varying data rates and distances.
  • Wireless does not always imply mobility.

Wireless Network Taxonomy

  • Infrastructure-based (e.g., APs):
    • Single-hop: Host connects directly to a base station (WiFi, WiMAX, cellular).
    • Multiple hops: Host relays through wireless nodes to connect to the Internet (mesh network).
  • No Infrastructure:
    • Single-hop: No base station, no Internet connection (Bluetooth, ad hoc networks).
    • Multiple hops: No base station, may relay to other wireless nodes (Piconet, MANET, VANET).

Wireless Links and Network Characteristics

  • Differences from wired links:
    • Decreased signal strength: Radio signals attenuate through matter.
    • Interference: Shared frequencies (e.g., 2.4 GHz) cause interference from other devices.
    • Multi-path propagation: Radio signals reflect off objects, arriving at the destination at different times.
  • Signal-to-Noise Ratio (SNR): Higher SNR improves signal extraction.
  • SNR versus BER tradeoffs:
    • Bit Error Rate (BER): Ratio of bit errors to total bits.
    • Increasing power increases SNR, decreasing BER.
    • Physical layer choice depends on SNR to meet BER requirements for throughput.
    • SNR can change with mobility, requiring dynamic physical layer adaptation.
  • Problems with multiple senders/receivers:
    • Hidden terminal problem: Nodes A and C cannot hear each other but interfere at node B.
    • Signal attenuation / Fading: Variation in signal strength.

Code Division Multiple Access (CDMA)

  • Each user is assigned a unique code (chipping sequence).
  • All users share the same frequency, using their chipping sequence to encode data.
  • Allows simultaneous transmission with minimal interference if codes are orthogonal.
  • Encoded signal = (original data) × (chipping sequence).
  • Decoding = inner-product of encoded signal and chipping sequence.

WiFi: 802.11 Wireless LANs

  • IEEE 802.11 wireless LAN is also known as WiFi.
  • Standards (e.g., 802.11b/g/n/ac/ax) use Carrier Sense Multiple Access with Collision Avoidance (CSMA/CA).
  • Supports both base-station (infrastructure) and ad-hoc modes.
  • 802.11 LAN Architecture:
    • Basic Service Set (BSS) (aka "cell") in infrastructure mode: wireless hosts + Access Point (AP).
    • Ad hoc mode: Hosts only.
    • Channels, association (Host-AP): 802.11b uses 2.4-2.485 GHz spectrum, 11 channels. AP admin chooses frequency; interference possible. Host association: Scans channels for beacon frames (passive) or sends probe request frames (active).
  • 802.11 MAC Protocol (CSMA/CA):
    • Goal: avoid collisions.
    • Sender: Senses channel idle for DIFS; transmits if idle. If busy, waits random backoff time. Retries if no ACK received.
    • Receiver: Returns ACK after SIFS (addresses hidden terminal problem).
    • Collision Avoidance: RTS-CTS exchange: Sender sends Request-To-Send (RTS); AP broadcasts Clear-To-Send (CTS). Other nodes defer transmission. Helps avoid collisions for long frames and addresses hidden terminal problem.
  • 802.11 Frame: Contains control, duration, addresses (1-4), sequence control, payload, and CRC fields.
  • Advanced Features in 802.11:
    • Rate Adaptation: APs dynamically change transmission rates based on SNR variations to maintain BER.
    • Power Management: Nodes can sleep and wake before beacon frames; beacons list pending AP-to-mobile frames.

Personal Area Networks (Bluetooth)

  • Typically < 10m diameter, cable replacement.
  • Ad hoc: No infrastructure.
  • Organized into a piconet (up to 8 active devices).
  • Master/Slaves: Slaves request permission to send; master grants.
  • 802.15.1: Uses 2.4-2.5 GHz band, up to 3 Mbps.

Cellular Networks: 4G/5G

  • Adopts Global System for Mobile (GSM) terminology.
  • Cell: Geographical region.
  • Base Station (BS): Analogous to 802.11 AP.
  • Mobile users attach via BS; air-interface between mobile and BS.
  • Mobile Switching Center (MSC): Connects cells to wired network, manages call setup and mobility.
  • 2G (Voice): GSM architecture (BTS, BSC, MSC). BSC controls multiple BTS.
  • 3G (Voice + Data): Universal Mobile Telecommunications Service (UMTS). Enables Internet access; requires full TCP/IP stack. New cellular data network operates in parallel with voice network.
    • 3G Core Networks: Serving GPRS Support Node (SGSN) (delivers datagrams to/from mobile in RAN) and Gateway GPRS Support Node (GGSN) (gateway to the Internet).
  • 4G: LTE (Long-Term Evolution):
    • All IP core: IP packets tunneled through the core IP network from base station to gateway.
    • No separation between voice and data: All traffic over IP core.
    • Architecture elements: User Equipment (UE), eNodeB (Base Station), Mobility Management Entity (MME), Home Subscriber Server (HSS), Serving Gateway (S-GW), PDN Gateway (P-GW).
    • Control Plane: New protocols for mobility management, security, authentication (MME, HSS).
    • Data Plane: Extensive tunneling (GPRS Tunneling Protocol, GTP) for mobility (S-GW, P-GW).
    • LTE Radio Access Network:
      • Downstream: OFDM (Orthogonal Frequency Division Multiplexing).
      • Upstream: FDM, TDM similar to OFDM.
      • Each mobile device allocated 0.5 ms time slots over channel frequencies.
      • Achieves 100s Mbps per device.
  • 5G Cellular Networks:
    • Goal: 10x peak bit rate, 10x lower latency, 100x traffic capacity over 4G.
    • 5G NR (New Radio): Two frequency bands (FR1: 450 MHz – 6 GHz, FR2: 24 GHz – 52 GHz, millimeter wave frequencies).
    • Not backwards-compatible with 4G.
    • MIMO: Multiple directional antennae.
    • Millimeter wave frequencies (FR2): Higher data rates, shorter distances; requires dense deployment of pico-cells.

4G/5G Compared to Wired Internet

  • Similarities: Edge/core distinction, global network of networks, widespread protocols (HTTP, TCP, IP), data/control plane separation, tunneling.
  • Differences: Different wireless link layer, mobility as a first-class service, user identity (SIM card), subscription-based model, "home network" vs. roaming, global access with authentication.

Mobility Management: Principles

  • Spectrum of mobility: From no mobility (within same wireless access network) to high mobility (moving between networks with ongoing connections).
  • Home network: Cellular provider network to which user subscribes.
  • Visited network: Network where mobile currently resides.

Routing to a Mobile Node (Mobile IP)

  • Home agent performs mobility functions for remote mobile.
  • Care-of-address (COA): Address in visited network.
  • Foreign agent: Entity in visited network performing mobility functions.
  • Registration: Mobile contacts foreign agent upon entering visited network; FA registers with HA, informing it of mobile's COA.
  • Mobile uses two addresses: permanent address (transparent to correspondent) and COA (used by HA for forwarding).
  • Mobility approaches:
    • Network (routers) handles it: Not scalable.
    • End-systems handle it:
      • **(a) Indirect Routing**: Correspondent addresses packets to mobile's permanent address. Home agent intercepts and tunnels packets to foreign agent's COA. FA decapsulates and forwards to mobile. Mobile replies directly to correspondent.
      • **(b) Direct Routing**: Correspondent gets mobile's care-of-address from home agent and sends packets directly. Overcomes triangle routing problem but is non-transparent to correspondent.
  • Mobile IP Standard (RFC 5944 for IPv4): Indirect Routing, Agent Discovery, Registration with Home Agent.
  • Agent Discovery: HAs/FAs advertise service via ICMP messages with H,F bits (home/foreign agent) and R bit (registration required).
  • Encapsulation and Decapsulation: Packets are sent within another packet (tunneling) to reach the mobile's COA.

Handling Mobility in Cellular Networks (GSM)

  • HLR (Home Location Register): Home network database with permanent phone #, profile, and current location.
  • VLR (Visitor Location Register): Visited network database with entry for each user currently in that network.
  • Indirect Routing to Mobile (GSM):
    1. Call routed to home network.
    2. Home MSC consults HLR for mobile's roaming number (in visited network).
    3. Home MSC sets up second leg to visited network's MSC.
    4. Visited MSC completes call via base station to mobile.
  • GSM Handoff with common MSC:
    • Goal: Route call via new base station without interruption.
    • Initiated by old BS (due to stronger signal from new BS or load balancing).
    • Mechanism: Old BS informs MSC; MSC sets up path to new BS; new BS allocates channel; new BS signals readiness; old BS tells mobile to handoff; mobile and new BS activate channel; mobile signals handoff complete; MSC reroutes call; old BS resources released.

Wireless and Mobility: Impact on Higher-Layer Protocols

  • Logically minimal impact: Best-effort service model, TCP/UDP can run over wireless.
  • Performance-wise:
    • Packet loss/delay due to bit errors, link-layer retransmissions, and handoff.
    • TCP misinterprets loss as congestion, causing unnecessary congestion window decrease.
    • Delay impairments for real-time traffic.
    • Limited bandwidth of wireless links.

Memory Aids

CDMA (Code Division Multiple Access)

Imagine a party where everyone speaks at once, but each uses a unique "code language." Only those knowing the code understand each other.

  • Encoding: Data bit × Chipping Sequence.
  • Decoding: Received Signal • Chipping Sequence.

CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance)

"Listen before you talk, and if you think someone else might talk simultaneously, politely wait, then confirm your message was heard." (Contrast with CSMA/CD where collisions are detected and transmission stops).

Mobile IP Roles (Home, Foreign, Care-of-Address)

  • HA: "Home Base" – knows where you should be and intercepts mail.
  • FA: "Friendly Agent" – temporary host in a foreign land helping with mail.
  • COA: "Current Operating Address" – your temporary forwarding address.

GSM Handoff

Like a relay race. The "old base station" hands off the "baton" (connection) to the "new base station" mid-run, so the "runner" (mobile) doesn't stop. Reasons: "Better connection, less crowded track."

Structured Study Guide Section

What to Remember

  1. Fundamental Differences: Understand wireless link characteristics vs. wired (signal attenuation, interference, multi-path propagation).
  2. CDMA Mechanics: Grasp unique chipping sequences for encoding/decoding, enabling simultaneous transmissions.
  3. WiFi (802.11) Protocol: Remember CSMA/CA (MAC) and RTS/CTS for collision avoidance (hidden terminal problem). Differentiate infrastructure vs. ad-hoc modes and scanning methods.
  4. Cellular Network Evolution: Recall 2G (voice), 3G (voice + data, parallel networks), 4G LTE (all-IP core, tunneling), and 5G goals/features (millimeter wave, MIMO).
  5. Mobile IP: Key entities (Home Agent, Foreign Agent, Correspondent) and addresses (Permanent Address, Care-of-Address). Understand indirect vs. direct routing and the registration process.
  6. Cellular Mobility: Recognize HLR and VLR roles in GSM for location management and how handoffs maintain call continuity.
  7. Impact on Higher Layers: Wireless is logically transparent, but performance issues (packet loss, TCP misinterpreting loss as congestion) are significant.

How Concepts Work

  • Signal Attenuation & BER/SNR: Signal strength diminishes with distance. Noise corrupts bits. Higher SNR means less BER. Networks dynamically adjust transmission rates (modulation) based on SNR to maintain BER and maximize throughput.
  • Hidden Terminal Problem: Nodes A and C can't hear each other but both transmit to B. Their transmissions collide at B. CSMA/CA with RTS/CTS resolves this: A sends RTS to B; B broadcasts CTS, silencing C. A then sends data.
  • CDMA Encoding/Decoding: Data bit (0 or 1) multiplied by a chipping sequence spreads the signal. Receiver multiplies incoming signal (sum of spread signals) by intended receiver's chipping sequence. Orthogonal codes recover intended signal, others appear as noise.
  • CSMA/CA (WiFi): Node senses channel. If idle for DIFS, transmits. If busy, waits random backoff. Expects ACK; no ACK triggers retransmission with increased backoff. RTS/CTS reserves channel for long frames.
  • Mobile IP Routing: Correspondent sends to mobile's permanent address. Home Agent (HA) intercepts. In indirect routing, HA tunnels packet to mobile's Care-of-Address (COA) (via Foreign Agent (FA)). FA decapsulates and forwards. Mobile replies directly. Direct routing has correspondent send directly to COA after discovery.

Common Student Mistakes

  • Confusing "Wireless" with "Mobility": Wireless is the medium; mobility is movement while connected.
  • Misunderstanding CSMA/CD vs. CSMA/CA: Ethernet uses Collision Detection (CD); wireless uses Collision Avoidance (CA) due to transmission difficulties.
  • TCP over Wireless Misinterpretation: TCP interprets wireless packet loss (bit errors) as congestion, unnecessarily reducing congestion window and impacting performance.
  • Incorrect CDMA Calculation: Forgetting to normalize the inner product result by the chipping sequence length during decoding.
  • Confusing HA and FA roles: Mixing up which agent is in the home vs. visited network and their specific functions.

Real-World Applications

  • WiFi (802.11): Ubiquitous for Internet access in homes, offices, public spaces. Essential for laptops, smartphones, smart home devices, IoT.
  • Bluetooth: Short-range personal area networks for wireless headphones, keyboards, mice, smartwatches, car systems.
  • Cellular Networks (4G/5G): Backbone of mobile communication.
    • 4G LTE: Powers smartphones for high-speed mobile internet, streaming, VoLTE.
    • 5G: Ultra-fast speeds, low latency, massive connectivity for autonomous vehicles, smart cities, AR/VR, advanced IoT.
  • Mobile IP: Principles of Home Agents and Care-of-Addresses are fundamental to IP-based mobility, though specific cellular protocols are more common in daily use.

Practice Questions & Answers

Question 1 (CDMA)

Consider a CDMA system where sender A wants to send data bit '0' and sender B wants to send data bit '1'.

  • Sender A's chipping sequence: (1, 1, -1, -1)
  • Sender B's chipping sequence: (1, -1, 1, -1)
  1. What is the encoded signal of sender A and sender B before they are added to the channel?
  2. What is the combined signal on the channel after both encoded signals are added?
  3. If a receiver wants to decode sender A's original data, what would be the decoded output? Show your working.

Model Answer 1:

  1. Encoded signal of sender A and sender B:
    • Data bit '0' is represented as -1.
    • Data bit '1' is represented as 1.

    Sender A (data bit '0' = -1):
    Encoded signal A = -1 × (1, 1, -1, -1) = (-1, -1, 1, 1)

    Sender B (data bit '1' = 1):
    Encoded signal B = 1 × (1, -1, 1, -1) = (1, -1, 1, -1)

  2. Combined signal on the channel:

    Combined signal = Encoded signal A + Encoded signal B
    Combined signal = (-1, -1, 1, 1) + (1, -1, 1, -1) = (0, -2, 2, 0)

  3. Decoding sender A's original data:

    Decoded output for A = (Combined signal • Sender A's chipping sequence) / Length of sequence
    Decoded output for A = ( (0×1) + (-2×1) + (2×-1) + (0×-1) ) / 4
    Decoded output for A = (0 - 2 - 2 + 0) / 4 = -4 / 4 = -1

    Since -1 represents data bit '0', sender A's original data bit is '0'.

Question 2 (WiFi & Mobility)

A user is moving through a large office building with multiple WiFi access points (APs) and maintains an active video call.

  1. Explain the process of how the user's device initially connects to an AP. Describe both passive and active scanning methods.
  2. How does the WiFi (802.11) MAC protocol aim to avoid collisions in this environment, and what specific mechanism helps with the "hidden terminal problem"?
  3. As the user moves from one AP's coverage to another, how does the network manage this "handoff" to ensure the video call continues without interruption within the same IP subnet?

Model Answer 2:

  1. Initial connection to an AP (Scanning methods):
    • Passive Scanning: Host listens for beacon frames from APs, which contain network info (SSID, MAC, rates). Host selects an AP, sends Association Request, receives Association Response.
    • Active Scanning: Host broadcasts Probe Request frames. APs respond with Probe Response frames. Host selects AP, sends Association Request, receives Association Response. Active scanning is often faster.
  2. Collision avoidance and the "hidden terminal problem" in 802.11:
    • CSMA/CA: Node senses channel. If idle for DIFS, transmits. If busy, waits random backoff. Expects ACK; no ACK triggers retransmission.
    • Hidden Terminal Problem: Nodes A and C can't hear each other but transmit to AP B, causing collision at B.
    • RTS/CTS exchange: Sender A sends RTS to AP B. AP B broadcasts CTS. Nodes like C hear CTS and defer transmission. A then sends data. This reserves the channel and avoids collisions.
  3. Handoff management within the same IP subnet:
    • Association/Reassociation: Device detects stronger signal from new AP and reassociates.
    • Self-learning Switches: Ethernet switches update forwarding tables to direct traffic to the new switch port where the device (by MAC address) is now reachable.
    • Seamless Connectivity: IP address remains constant, routing tables adapt quickly, maintaining ongoing connections like video calls without interruption.
