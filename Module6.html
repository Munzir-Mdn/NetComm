The Link Layer

Foundations of Network Communication

Summary and Basic Definitions

The Link Layer (Layer 2 of the OSI model) ensures reliable data transfer between physically adjacent network nodes. It encapsulates network-layer datagrams into frames, handling local data delivery, error detection/correction, multiple access protocols, and addressing within a local network segment. This guide explores its principles, services, and technologies, including applications in data center networking and web request processing.

Basic Definitions

Host:

An end-system device (e.g., computer, smartphone) on a network.

Router:

A network device that forwards data packets between networks, operating at Layer 3.

Communication Channels (Links):

Physical pathways connecting adjacent nodes (wired, wireless, LANs).

Layer-2 Packet (Frame):

Data unit at the link layer, encapsulating a network-layer datagram.

Data-link Layer Responsibility:

Transferring a frame from one node to a physically adjacent node over a link.

Link Layer Adapter (NIC):

Hardware implementing the link and physical layers (e.g., Ethernet card).

Key Concepts

Link Layer Services

  • Error Detection and Correction: Mechanisms to identify and potentially fix corrupted bits.
    • Flow Control: Pacing between adjacent sending/receiving nodes.
    • Error Detection: Receiver detects errors, may request retransmission or drop.
    • Error Correction: Receiver identifies and corrects bit errors.
    • Half-duplex / Full-duplex: Bidirectional, but not simultaneously / simultaneous bidirectional transmission.
  • Sharing a Broadcast Channel: Multiple Access Protocols govern sharing.
  • Link Layer Addressing: Using MAC addresses for local identification.

Error Detection and Correction Techniques

  • Single Bit Parity: Detects single bit errors.
  • Two-dimensional Bit Parity: Detects and corrects single bit errors.
  • Cyclic Redundancy Checks (CRC): Powerful, detects burst errors (Ethernet, 802.11).
  • Internet Checksum: (Transport layer) Detects "flipped bits".

Multiple Access Protocols Categories

Channel Partitioning Protocols:

  • TDMA: Fixed-length time slots.
  • FDMA: Channel spectrum divided into frequency bands.
  • CDMA: Unique code for each node, allowing simultaneous transmission.

Random Access Protocols:

  • Slotted ALOHA / Pure ALOHA: Transmit at slot start / transmit immediately; retransmit on collision.
  • CSMA: "Listen before transmit."
  • CSMA/CD: (Wired LANs) Detect collisions, abort, backoff, retransmit.
  • CSMA/CA: (802.11 wireless) Collision avoidance.

Taking Turns Protocols:

  • Polling: Master invites slave nodes.
  • Token Passing: Token grants transmission rights.

Local Area Networks (LANs)

Link Layer Addressing (MAC Addresses):

  • MAC Address: 48-bit physical address, burned into NIC ROM, for local frame transfer.
  • OUI: First 3 bytes (manufacturer).
  • Vendor Assigned: Last 3 bytes.
  • MAC Address Portability: Flat address, portable.
  • IP Address: Hierarchical, network-wide, not portable.

Address Resolution Protocol (ARP):

  • Determines MAC address given its IP address.
  • ARP table: IP-to-MAC mapping with TTL.
  • ARP Query: Broadcasts request.
  • ARP Reply: Target responds with MAC.

Ethernet:

  • Dominant wired LAN (IEEE 802.3 CSMA/CD).
  • Connectionless, Unreliable: Higher layers recover dropped data.
  • Frame Structure: Preamble, Dest/Source MAC, Type, Data, CRC.
  • Physical Topology: Now predominantly switched (separate collision domain per link).

Link-Layer Switches:

  • Active role: store-and-forward Ethernet frames.
  • Transparent, Plug-and-play, Self-learning: Build forwarding table.
  • Filtering: Drops frames if destination on same segment.
  • Forwarding: Sends to specific output interface.
  • Flooding: If unknown, forwards on all interfaces except arrival.
  • Supports multiple simultaneous transmissions.

Virtual Local Area Networks (VLANs):

  • Define multiple virtual LANs over single physical infra.
  • Port-based, Traffic Isolation, Dynamic Membership.
  • 802.1Q protocol: Uses trunk links to carry VLAN ID.

Data Center Networking

  • Hundreds of thousands of hosts.
  • Challenges: Massive clients, load management, bottleneck avoidance.
  • Hierarchical Topology: Border router -> Load balancer -> Access router -> Tier-1/2 switches -> TOR switches -> Server racks.
  • Trends: Rich interconnection for throughput and redundancy.

A Day in the Life of a Web Request (Synthesis)

  1. DHCP: Laptop connects, gets IP, router IP, DNS server IP (broadcast request).
  2. ARP: Client needs MAC of first-hop router (ARP query broadcast, router replies).
  3. DNS: Client queries DNS server to resolve www.google.com to an IP.
  4. TCP Connection: Client opens TCP socket to web server (3-way handshake).
  5. HTTP Request/Response: HTTP request sent over TCP, routed to web server; server responds, routed back.

Memory Aids

CRC Calculation Steps (DRGR):

  • Data bits
  • Append r zeros
  • Agree on Generator polynomial (G)
  • Divide (modulo 2) to get Remainder
  • Transmit D followed by R

MAC Address vs. IP Address:

  • MAC: Social Security Number – unique, flat, burned-in, local.
  • IP: Postal Address – hierarchical, network-wide, not portable.

MAC Protocol Categories (CRaT):

  • Channel Partitioning
  • Random Access
  • Taking Turns

CSMA/CD Analogy:

"The Polite Conversationalist" – listen before speaking, if others speak simultaneously, stop and wait before trying again.

Switch Learning:

  • Source = Learn: Record source MAC and port.
  • Destination = Forward/Flood: Look up dest MAC; if found, forward; if not, flood.

Structured Study Guide

What to Remember

  • The Link Layer's role in local, hop-by-hop data transfer.
  • Error detection vs. correction mechanisms (parity, CRC).
  • Necessity and types of multiple access protocols for shared media.
  • Distinction and relationship between MAC and IP addresses, and ARP's role.
  • Ethernet characteristics: frame structure, connectionless/unreliable, CSMA/CD.
  • Link-layer switches: forwarding table building (self-learning), filtering/forwarding.
  • VLANs: purpose, benefits (isolation, dynamic membership), and 802.1Q protocol.
  • The step-by-step process of a network request (e.g., web request) from application to link layer and back.

How Concepts Work

  • Error Detection: Sender appends redundant bits. Receiver compares calculated bits with received ones.
  • Multiple Access Protocols: Dictate transmission on shared medium. Channel partitioning pre-allocates; Random Access allows contention; Taking Turns ensures orderly access.
  • MAC Addressing: Unique hardware IDs for local frame delivery by switches.
  • ARP Operation: Broadcasts query for local MAC given IP; target replies. Mappings cached.
  • Switch Learning and Forwarding: Switches learn MACs from source addresses. Forward by looking up destination MAC; flood if unknown.
  • VLANs: Logically segment a physical network for isolation. 802.1Q tags frames on trunk links.

Common Student Mistakes

  • Confusing IP and MAC Addresses: IP is logical/hierarchical/network-layer; MAC is physical/flat/link-layer.
  • Misunderstanding Collision Domains: Hubs = one large; Switches = separate per port.
  • Incorrect CRC Calculations: Careful modulo 2 division.
  • Overlooking Ethernet's Unreliability/Connectionlessness: Higher layers (TCP) handle reliability.
  • Not knowing when to use ARP: For *local* device MAC addresses when IP is known.

Effective Study Tips

  • Draw Diagrams: Visualize protocols, frame paths.
  • Practice CRC Calculations: Manually work through examples.
  • Compare and Contrast: Use tables for protocols, hubs vs. switches, IP vs. MAC.
  • Trace Web Requests: Understand step-by-step (DHCP -> ARP -> DNS -> TCP -> HTTP).
  • Review Frame Structures: Memorize Ethernet frame fields.

Real-World Applications

  • Ethernet: Backbone of wired LANs.
  • Wi-Fi (802.11): Ubiquitous wireless LAN (CSMA/CA).
  • Cellular Networks: CDMA for 3G/4G wireless.
  • Data Centers: Complex link layer for traffic, load balancing, redundancy.
  • Everyday Web Browsing: Link layer moves frames, uses ARP, and switches.

Exercise 6.1: CRC Calculation (Sender)

If the data to be sent is 101110 and the CRC technique is used with r = 3 and G = 1010.

  1. What is the value of R?
  2. What is the data that will be sent?

Model Answer:

Data D = 101110, r = 3, Generator G = 1010.

  1. Append r=3 zeros to D: 101110000.
  2. Perform Modulo 2 Division of 101110000 by 1010.
101110000
1010
-----
000110000
   1010
   ----
   0110000
      1010
      ----
      01100
         1010
         ----
         0110

The remainder is 0110. Since r=3, we take the last 3 bits if the result is longer than r bits and the leading bit is 0. The remainder is 110.

a) R = 110

b) Transmitted data: D concatenated with R = 101110110.

  2. Is the data error?

Received data D_prime = 1011101100, Generator G = 10100 (5 bits).

Conclusion

The Link Layer is fundamental to local network communication, ensuring reliable data transfer between adjacent nodes. It provides essential services like error detection and correction for data integrity and employs various multiple access protocols for efficient shared channel management. Link layer addressing, using MAC addresses, is vital for local frame delivery, working with ARP to map IP addresses to MAC addresses. Ethernet, the dominant wired LAN technology, operates using connectionless, unreliable principles with CSMA/CD. Link-layer switches improve network efficiency by intelligently forwarding frames and creating dedicated collision domains, while VLANs enable flexible network segmentation. A thorough understanding of the Link Layer, from its core principles to its application in data centers and web requests, is crucial for comprehending network communication.
